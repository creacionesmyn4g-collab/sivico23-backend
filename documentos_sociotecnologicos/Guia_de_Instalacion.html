Guía de Instalación

SIVICO23 - Sistema Integral de Vigilancia Comunitaria

Aprobado para Voceros de Salud
Nota Socio-Tecnológica: Esta guía está diseñada paso a paso para que cualquier miembro de la comunidad con conocimientos básicos de informática pueda levantar el sistema localmente en su Ambulatorio o Base de Misiones.
1

Requisitos Previos

Prepare el entorno antes de instalar SIVICO23. Es importante tener instaladas las herramientas correctas para evitar problemas al iniciar el backend y el frontend.

  • Node.js 18+ y npm 9+.
  • Expo CLI: npm install -g expo-cli.
  • Git: clonar el repositorio completo.
  • Base de datos PostgreSQL / Supabase: crear el proyecto y obtener la URL de conexión.
2

Instalar y Configurar el Backend

Desde la carpeta backend, instale dependencias y configure la conexión a PostgreSQL.

cd backend
npm install

Luego cree un archivo .env con al menos estas variables:

DATABASE_URL=postgresql://usuario:clave@host:puerto/base
JWT_SECRET=una_clave_segura

Opcionalmente puede usar variables individuales si no utiliza DATABASE_URL:

DB_HOST=localhost
DB_PORT=5432
DB_USER=sivico23_user
DB_PASSWORD=sivico23_password
DB_NAME=sivico23_db

Arranque el backend con:

npm run dev
3

Configurar el Punto de Acceso (Frontend)

Antes de compilar la App, debe indicar la URL de su backend y asegurarse de tener instaladas las dependencias del frontend.

cd frontend
npm install

Luego abra frontend/src/utils/constants.js y actualice la variable API_BASE_URL.

// Para un servidor en la nube
export const API_BASE_URL = 'https://sivico23-backend.onrender.com';
// Para pruebas en red local Android
export const API_BASE_URL = 'http://192.168.x.x:3000';
4

Compilar el APK (EAS Build)

Para generar el archivo instalable para los teléfonos de los voceros, usamos Expo Application Services (EAS):

cd frontend
npx eas build --platform android --profile apk

Esto generará un enlace de descarga seguro para el archivo .apk listo para instalar.

5

Desarrollo Local y Pruebas

Si desea hacer cambios rápidos o pruebas sin internet, puede usar el servidor local con el túnel de Expo:

npx expo start --tunnel

Escanee el código QR con la app Expo Go en su teléfono.

6

Verificación y Cuentas Iniciales

Compruebe que el backend y el frontend se hayan conectado correctamente con estos pasos:

  • Abra el navegador en http://localhost:3000/api/health para verificar el backend.
  • Use cuentas preconfiguradas para pruebas: admin@sivico23.ve, vocero@sivico23.ve y medico@sivico23.ve.
  • Valide que los datos de sectores, pacientes y registros se sincronizan desde la app móvil.
