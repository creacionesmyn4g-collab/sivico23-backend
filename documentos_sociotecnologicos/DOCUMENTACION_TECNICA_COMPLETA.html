1. Descripción del Proyecto

SIVICO23 es un sistema de gestión de salud comunitaria fundamentado en la Tercera Forma Normal (3FN). A diferencia de sistemas planos, SIVICO23 utiliza tablas maestras para garantizar que no existan redundancias ni errores de integridad en la información de la Parroquia 23 de Enero.

3. Modelo de Base de Datos (17 Tablas)

Defensa Técnica: El sistema ha evolucionado de un modelo básico a uno de **17 tablas** para cubrir necesidades epidemiológicas avanzadas. Los catálogos de Vacunas, Discapacidades y Jornadas son entidades independientes, lo que garantiza el cumplimiento de la **3FN** en toda la lógica de salud pública.

Estructura de las 17 Tablas

Núcleo Operativo

  1. roles: Perfiles de acceso.
  2. usuarios: Personal del sistema.
  3. sectores: Georreferenciación.
  4. pacientes: Población civil monitorizada.
  5. registros: Encuentros médicos.
  6. tratamientos: Diagnósticos (CIE-10).
  7. medicamentos: Prescripción farmacológica.

Salud Pública & Especialidades

  1. categorias_cie10: Clasificación OMS.
  2. patologias_cie10: Glosario CIE-10 completo (7.000+ diagnósticos).
  3. cat_vacunas_mpps: Catálogo oficial de inmunización.
  4. registro_vacunacion: Historial PAI multi-dosis.
  5. cat_discapacidades: Tipificación CONAPDIS.
  6. paciente_discapacidades: Registro de certificados.

Comunidad & Emergencias

  1. alertas_emergencia: Señales SOS con GPS.
  2. jornadas_salud: Planificación comunitaria.
  3. notificaciones_usuarios: Centro de alertas.
  4. auditoria_pacientes: Trazabilidad de cambios.

4. Diccionario de Datos (17 Tablas)

Justificación de Normalización (3FN): Para cumplir con la Tercera Forma Normal, se han eliminado las dependencias transitivas. Los atributos que dependían de otros atributos no primarios (como el nombre del sector o el nombre del rol) se han movido a sus propias entidades maestras, vinculándose mediante IDs numéricos.

    5. Justificación Socio-Tecnológica (El "Por qué")

    5.1 Necesidad de la Solución

    En la Parroquia 23 de Enero, la salud comunitaria dependía históricamente de registros manuales y "fichas de cartón" que se perdían, se deterioraban o no permitían un análisis rápido. SIVICO23 nace para transformar el **dato pasivo** en **acción epidemiológica**. La necesidad tecnológica radica en centralizar la información para detectar brotes en tiempo real, asegurar que ningún niño pierda su esquema de vacunación y georreferenciar las vulnerabilidades sociales.

    5.2 Elección del Lenguaje y Herramientas

    • React Native / Expo: Se eligió por su capacidad **multiplataforma**. Los voceros de salud usan diversos dispositivos Android. Esta tecnología permite desplegar una App nativa y fluida sin necesidad de programar dos veces, facilitando el acceso tecnológico a la comunidad.
    • Node.js (JavaScript): Es el lenguaje de la web moderna. Su alta concurrencia permite que cientos de voceros automaticen y sincronicen datos simultáneamente sin colapsar el servidor, garantizando que el sistema sea **ligero y escalable**.
    • PostgreSQL: A diferencia de bases de datos simples, PostgreSQL ofrece una **integridad relacional estricta**. Esto es vital en medicina: un tratamiento no puede existir sin un paciente, y una vacuna no puede quedar "en el aire". El uso de SQL garantiza que la historia clínica sea inviolable y coherente.

    5.3 Impacto Social

    El uso de estas tecnologías permite el **Empoderamiento Tecnológico** de la comunidad. Ya no dependen de un ente externo para saber cuántos hipertensos hay en su calle; el sistema les devuelve el poder de la información para gestionar sus propios recursos y medicinas ante las autoridades de salud.

    6. Stack Tecnológico Detallado

    📱 Frontend (Móvil)

    • React Native / Expo: Framework para ejecución nativa en Android e iOS.
    • React Navigation: Gestión de pilas y flujo de pantallas.
    • Axios: Cliente HTTP con interceptores para manejo de tokens.
    • AsyncStorage: Persistencia local de datos (Caché y Offline).
    • Catálogo Curado: Frontend incluye catálogo optimizado de 50 patologías principales + 138 medicamentos únicos para uso offline, complementado con búsqueda dinámica al backend CIE-10 completo (7.000+ diagnósticos).

    ⚙️ Backend & DB

    • Node.js & Express: Servidor de API REST bajo arquitectura MVC.
    • PostgreSQL 16: Base de datos relacional robusta.
    • PG-Pool: Gestión de conexiones concurrentes optimizada.
    • Bcrypt: Encriptación de nivel bancario para credenciales.

    6. Lógica de Persistencia y Sincronización

    SIVICO23 implementa una arquitectura de "Primero Local, Luego Nube" para garantizar operatividad en zonas de baja conectividad:

    Sincronización Inteligente: Cuando se detecta conexión, el sistema "re-hidrata" la base de datos central mediante un proceso de reconciliación que evita duplicados y asegura que los registros locales se consoliden con el historial global del paciente.

    7. Arquitectura de Catálogos Médicos

    SIVICO23 implementa una estrategia híbrida de catálogos para optimizar rendimiento y funcionalidad:

    📱 Catálogo Frontend (Offline-First)

    • 50 Patologías Principales: Curaduría manual de diagnósticos más comunes en contexto comunitario venezolano.
    • 138 Medicamentos Únicos: Formulaciones esenciales con dosificación y vías de administración.
    • Persistencia Local: AsyncStorage garantiza funcionamiento sin conexión a internet.
    • Optimización UX: Búsqueda rápida y autocompletado sin dependencias de red.

    ☁️ Catálogo Backend (CIE-10 Completo)

    • 7.000+ Diagnósticos: Cobertura total de la Clasificación Internacional de Enfermedades (CIE-10).
    • Búsqueda Dinámica: API de búsqueda en tiempo real para diagnósticos especializados.
    • Actualización Continua: Scripts automatizados mantienen sincronización con estándares OMS.
    • Integridad Médica: Códigos oficiales garantizan precisión diagnóstica.
    Beneficios de la Arquitectura Híbrida: El catálogo curado del frontend asegura usabilidad offline, mientras que el backend completo permite diagnósticos especializados y mantiene la integridad médica profesional.

    8. Mapa Interactivo y Estadísticas por Sector

    El sistema incluye un módulo de cartografía digital (`MapaSectores`) que permite visualizar la situación sanitaria de la Parroquia 23 de Enero:

    • Zonificación: Representación visual de los sectores oficiales.
    • Estadísticas en Tiempo Real: Al seleccionar un sector, el Dashboard filtra automáticamente patologías top, población por sexo y distribución de edades de esa zona específica.

    8. Monitoreo de Emergencias (SOS)

    Módulo crítico para la respuesta inmediata ante eventos de salud:

    Flujo SOS: El usuario activa la alerta con un solo toque; el sistema captura las coordenadas GPS y las envía al panel de control central (visto por administradores y médicos) para coordinar la atención.

    9. Integridad de Datos y Deduplicación

    Para mantener la calidad de la información epidemiológica, el backend implementa procesos automáticos:

    • Deduplicación en Caliente: Al arrancar el servidor, se ejecutan scripts de limpieza que unifican historiales de pacientes registrados bajo la misma cédula por error.
    • Blindaje SQL: Uso de transacciones atómicas y `UPSERT` para evitar colisiones durante sincronizaciones masivas.

    10. Análisis Epidemiológico Avanzado (v4.0)

    El Dashboard v4.0 introduce motores de cálculo en tiempo real:

    • Analítica PAI: Calcula la cobertura de vacunación cruzando edad vs. dosis aplicadas.
    • Detección de Niños en Riesgo: Identifica infantes con esquemas incompletos para su edad.
    • Vigilancia de Discapacidad: Censo georreferenciado de personas con necesidades especiales y carnet CONAPDIS.

    11. Programa Ampliado de Inmunización (PAI)

    Lógica de Negocio: SIVICO23 automatiza el control de dosis. Por ejemplo, al registrar una "Pentavalente", el sistema sabe que el infante requiere 5 dosis en intervalos específicos, permitiendo generar alertas de "Mora de Vacunación".
    • Buscador PAI: Interfaz simplificada para encontrar niños y ver su "Tarjeta de Vacunación Digital".
    • Carga de Dosis: Registro de fecha, lote y responsable de la aplicación.

    12. Módulo de Discapacidad

    Cumpliendo con la Ley para Personas con Discapacidad:

    • Clasificación CONAPDIS: Registro de tipos de discapacidad (Visual, Auditiva, Motora, etc.).
    • Validación de Certificados: Campo específico para el número de certificado oficial, facilitando la gestión de ayudas sociales en el sector.

    13. Seguridad e Integridad

    • Consultas Parametrizadas: Blindaje total contra Inyección SQL ($1, $2).
    • Bcrypt: Las contraseñas nunca se guardan en texto plano, solo el hash.
    • Integridad Referencial: Uso de ON DELETE RESTRICT para evitar borrar un sector que tiene pacientes registrados.
    • JWT: Autenticación persistente con tokens de larga duración (7 días).
