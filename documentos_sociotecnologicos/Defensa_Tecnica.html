Guía de Defensa Técnica - SIVICO23

Documento de apoyo para la exposición oral y técnica de la aplicación de vigilancia sociocomunitaria.

Esta guía explica el idioma del código, los comandos usados, las decisiones tecnológicas y las respuestas que el jurado espera.

1. Tecnologías principales

Frontend

SIVICO23 usa React Native con Expo para la app móvil. No se usó Java puro porque React Native permite un solo código base para Android e iOS, y reduce tiempo de desarrollo para el equipo.

Ventajas:

  • Multiplataforma con un solo proyecto.
  • Actualizaciones más rápidas.
  • Expo facilita pruebas en dispositivos reales sin compilar APKs cada vez.

Backend

El servidor se implementó en Node.js usando Express. Esto permite APIs REST ligeras y alto rendimiento con pocas líneas de código.

Por qué Node.js:

  • Asincronía nativa con async/await para consultas de base de datos.
  • Gran ecosistema de librerías.
  • Es el lenguaje de la web, ideal para conectar la app móvil con PostgreSQL.

2. Comandos clave y su razón

Backend

  • cd backend — Moverse al proyecto del servidor.
  • npm install — Instala dependencias Node.js necesarias.
  • npm run dev — Inicia el servidor con nodemon para recarga automática durante el desarrollo.
  • npm start — Inicia el backend en modo producción.

Frontend

  • cd frontend — Cambia al proyecto de la app móvil.
  • npm install — Instala librerías de React Native y Expo.
  • npx expo start — Arranca el servidor de desarrollo Expo.
  • npx expo start --tunnel — Permite conectar teléfonos reales aunque el backend esté en la PC.
  • npx eas build --platform android --profile apk — Genera un APK instalable para pruebas de campo.

3. Lenguajes usados y por qué

JavaScript

Lenguaje único para el frontend y backend. Facilita la integración de datos y la lógica compartida.

React Native

Framework que compila a componentes nativos de Android e iOS sin escribir Java/Kotlin.

SQL

Usado en PostgreSQL para definir el esquema, consultas, índices y vistas del sistema de salud.

React Native se eligió porque el proyecto requiere vigilancia comunitaria móvil con disponibilidad rápida en múltiples dispositivos Android, y Expo reduce tiempos de despliegue frente a Java nativo.

5. Base de datos usada y su justificación

La base de datos es PostgreSQL, version 16 compatible. Se eligió por su robustez, soporte para integridad referencial y cumplimiento con esquemas sanitarios donde los datos deben ser consistentes.

Por qué PostgreSQL

  • Mejor integridad relacional y transaccional.
  • Soporte para JSONB y funciones avanzadas.
  • Seguridad y estabilidad para aplicaciones críticas.
  • Más adecuado que MySQL para datos clínicos normalizados.

7. Palabras usadas en la programación y su significado

Conceptos Frontend

  • API_BASE_URL: dirección del servidor donde se hacen las peticiones.
  • AsyncStorage: memoria local del teléfono para datos offline.
  • axios: biblioteca para hacer GET, POST, PUT, DELETE.
  • formik: gestiona formularios y envíos.
  • yup: valida datos antes de enviarlos al servidor.

Conceptos Backend

  • express.json(): convierte JSON de la petición en objeto JavaScript.
  • jwt.verify(): valida tokens para autorizar solicitudes.
  • bcrypt.hash(): genera hash seguro de contraseñas.
  • pool.query(): ejecuta consultas SQL con el cliente PostgreSQL.
  • REFERENCES: fuerza integridad entre tablas relacionadas.

8. Ejemplo de programación en React Native

En SIVICO23 el desarrollo móvil se basa en componentes funcionales de React, usando useState, useEffect y composición. No se usa herencia clásica; en lugar de eso, se reutilizan componentes y se pasan datos mediante props.

Herencia vs composición

React Native no se programa con herencia clásica como en Java. El patrón preferido es composición, creando componentes pequeños y combinándolos. Por ejemplo, una tarjeta de paciente se puede reutilizar en RegistroScreen y HistorialScreen sin heredar de una clase base.

La aplicación usa componentes independientes que se combinan, no clases que extienden otras clases. Esto mejora la mantenibilidad y la reutilización.

Este enfoque responde a preguntas como:

  • ¿Se usa herencia? No, se usa composición de componentes.
  • ¿Qué palabras clave se utilizaron? useState, useEffect, props, return, async/await, import, export default.
  • ¿Por qué no usar clases? Las funciones son más sencillas, ocupan menos código y trabajan mejor con hooks, que son el estándar moderno de React.

9. Preguntas frecuentes del jurado y respuestas

¿Por qué no Java nativo?

React Native permite mantener una sola base de código, lo cual es ideal cuando el equipo tiene menor tiempo y se requiere velocidad de entrega. Java nativo implica desarrollar y mantener Android y iOS por separado.

¿Por qué PostgreSQL?

Porque ofrece transacciones fuertes, claves foráneas, vistas y triggers. Estas capacidades son críticas para un sistema de salud que necesita integridad y auditoría de datos.

¿Qué hace el módulo offline?

Almacena registros en AsyncStorage y los sincroniza cuando hay conexión. Esto permite a los voceros trabajar en zonas sin internet y subir datos desde la base de misiones.

¿Cómo se asegura la contraseña?

Se usa bcrypt con 10 rondas de hash. Las contraseñas nunca se guardan en texto plano y se comparan con bcrypt.compare().

¿Qué es 3FN?

La Tercera Forma Normal es una técnica para eliminar redundancias. En SIVICO23 se separan datos de pacientes, usuarios, sectores, patologías, vacunas y notificaciones en tablas diferentes.

¿Qué archivos debe revisar el jurado?

  • backend/database.sql — Esquema y triggers.
  • backend/server.js — Rutas, autenticación y lógica REST.
  • frontend/src/utils/constants.js — Configuración de servicios.
  • frontend/App.js — Flujo de navegación y arranque.
